버그 수정 절차:

1. 증상을 한 문장으로 정리한다.
2. 관련 파일과 기존 패턴을 찾는다.
3. 가능하면 실패 테스트를 먼저 만든다.
4. 원인을 확인한다.
5. 가장 작은 수정만 적용한다.
6. 좁은 테스트부터 실행한다.
7. 검증 명령은 README, manifest, Makefile, scripts에서 확인한다.
8. 필요하면 타입체크/린트/빌드를 실행한다.
9. 완료 보고에 변경/검증/위험을 적는다.

금지:
- 원인 모른 채 우회 코드 작성
- 실패 테스트 삭제
- 관련 없는 리팩터링
- Mock으로 실제 문제 숨기기
