# AI_CONTEXT.md

프로젝트별로 채워 넣을 수 있는 맥락 문서의 기준이다. 이 파일 자체는 전역 설치 대상이 아니다.

## 사용 원칙

- 전역 규칙에 넣기 어려운 프로젝트 고유 정보를 여기에 둔다.
- 실제 값이 없으면 빈 placeholder를 남기지 말고 해당 항목을 삭제한다.
- 명령을 추측해서 쓰지 않는다. README, manifest, lockfile, Makefile, scripts에서 확인된 명령만 적는다.

## 권장 항목

- 앱 유형과 주요 런타임
- 주요 진입점과 디렉터리 역할
- 재사용해야 하는 API, 상태 관리, 에러 처리, 로깅, 테스트 유틸
- 수정 주의 영역: 인증, 권한, 결제, DB migration, 데이터 삭제, production secret
- 검증 명령: 테스트, 단일 테스트, 타입체크, 린트, 빌드
- 배포 경계: 자동화 가능 작업과 반드시 사용자 승인이 필요한 작업

## 작성 규칙

- 경로는 실제 존재하는 파일이나 디렉터리만 적는다.
- 명령은 현재 checkout에서 실행 가능한 형태로 적는다.
- 오래된 정보는 보존하지 말고 삭제한다.
- 문서가 비어 있거나 부정확하면 Claude/Codex가 repo에서 다시 확인하도록 남긴다.

## 금지

- 동일 기능 중복 구현 지시
- 존재하지 않는 파일 경로
- 검증되지 않은 빌드/테스트 명령
- 보안/권한 경계를 약화하는 프로젝트 예외
