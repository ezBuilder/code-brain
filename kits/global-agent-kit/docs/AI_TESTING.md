# AI_TESTING.md

전역 규칙 키트와 프로젝트 작업의 검증 기준이다.

## 명령 발견

- README, package manifest, lockfile, Makefile, scripts, CI workflow에서 검증 명령을 찾는다.
- 찾지 못한 명령은 추측해서 실행하지 않는다.
- 프로젝트별 `AI_CONTEXT.md`가 있으면 거기에 적힌 실제 명령을 우선한다.

## 검증 우선순위

1. 변경 파일에 가장 가까운 테스트
2. 관련 기능 단위 테스트
3. 타입체크
4. 린트
5. 전체 테스트
6. 빌드
7. 설치 또는 배포 dry-run

## 원칙

- 버그 수정은 가능하면 실패 테스트나 재현 절차를 먼저 만든다.
- 테스트를 삭제하거나 약화하지 않는다.
- Mock은 외부 경계에만 사용한다.
- 실제 문제를 Mock으로 숨기지 않는다.
- 검증을 못 하면 이유와 대신 확인한 내용을 완료 보고에 적는다.

## 이 키트의 기본 검증

```bash
./scripts/validate.sh
```

`make validate`는 같은 검증을 실행하는 편의 target이다.
