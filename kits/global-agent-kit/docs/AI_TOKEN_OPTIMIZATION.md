# AI_TOKEN_OPTIMIZATION.md

전역 규칙 키트의 토큰/컨텍스트 최적화 기준이다.

## 원칙

- `rules/CLAUDE.md`와 `rules/AGENTS.md`는 매 세션 로드될 수 있으므로 짧게 유지한다.
- 코드에서 알 수 있는 스택, 라이브러리 목록, 일반 설명은 전역 규칙에 넣지 않는다.
- 자주 바뀌는 세부 정책은 `docs/`에 두고 필요할 때만 읽는다.
- 전역 규칙을 자주 바꾸면 프롬프트 캐시 효율이 떨어질 수 있으므로 안정된 내용만 넣는다.

## 도구 출력

- 전체 파일보다 필요한 줄 범위를 읽는다.
- 검색은 디렉터리, 파일 확장자, 패턴을 좁힌다.
- 긴 `find`, `tree`, broad `rg` 출력은 피한다.
- Code Brain이 있는 프로젝트에서는 검색/긴 출력 요약 경로를 우선한다.
- RTK 같은 외부 절약 도구는 설치가 확인된 프로젝트에서만 사용한다. 전역 규칙은 이를 필수 의존성으로 두지 않는다.

## 세션 관리

- 작업 단위로 세션을 분리한다.
- 긴 대화는 `/compact` 또는 새 세션 전환을 고려한다.
- 토큰/비용 확인이 필요하면 Claude Code의 `/cost` 같은 내장 사용량 명령을 활용한다.

## Ignore 기준

프로젝트별 `.claudeignore`에는 빌드 산출물과 대용량 생성물을 넣는다.

기본 제외 후보:

- `node_modules/`, `.venv/`, `.next/`, `.nuxt/`
- `dist/`, `build/`, `coverage/`, `.cache/`
- `.ai/cache/`, `.ai/runtime/.venv/`
- `*.min.js`, `*.map`, `.DS_Store`

lockfile은 패키지 매니저와 dependency 확인에 유용하므로 기본 제외하지 않는다.
